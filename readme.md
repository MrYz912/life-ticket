# 🎟️ 人生票夹 (Life Ticket Folder)

> **每一个光影瞬间，都值得一张永不褪色的纪念票。**

## 🌟 项目亮点：全链路 AI 开发实验

本项目是一个**全实验性**的软件工程尝试。我们挑战了在**不手写一行代码**的情况下，如何利用当前最尖端的 AI 模型协作构建出一个高可用、高审美的移动端应用。

- **100% AI 生成**：全量业务逻辑、UI 组件及底层架构均由 AI 自动构建。
- **多模型协同**：
  - **架构与逻辑**：由 **TRAE (Solo Mode)** 驱动，深度调用 **MiniMax-2.5** 与 **GLM-5.0** 协同编写。
  - **指令工程**：通过 **Gemini 3 Pro** 对原始需求进行深度结构化优化，确保代码生成的精准度。
- **人机角色**：人类作为“产品经理”与“架构评审”，AI 承担“全栈工程师”职责。

------

## ✨ 核心功能

- **🎬 智能录入**：快速搜索记录电影名称、观影时间、影院、价格、人数、个人感悟。
- **🎫 纪念票生成**：内置精致模版，根据录入信息自动排版，生成具有电影感、复古感的数字化纪念票。
- **📁 个人票夹**：结构化展示你的观影历史，追溯你的光影人生。
- **🖼️ 一键留存**：支持将生成的票根以高清晰度图片格式导出，适配社交媒体分享或手机相册永久留存。

------

## 🛠️ 技术栈与 AI 工作流

### 1. 开发环境

- **编辑器**: ByteDance Trae (Solo Mode)
- **目标平台**: React Native 适配 IOS 和 Android

### 2. AI 协同流程图

本项目采用了 **“需求 -> 提示词增强 -> 代码注入”** 的闭环：

1. **需求打磨**：使用 **Gemini 3 Pro** 将模糊的创意转化为结构化的 `System Prompt` 和 `Feature Specs`。
2. **代码实现**：利用 **TRAE** 的上下文感知能力，交替使用 **MiniMax-2.5** (擅长长文本逻辑) 和 **GLM-5.0** (擅长中文语境和UI细节) 进行代码编写。
